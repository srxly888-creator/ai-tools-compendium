# 贡献指南

感谢你对 AI Tools Compendium 的兴趣！我们欢迎任何形式的贡献。

## 🤔 如何贡献

### 1. 报告问题

如果你发现：
- 报告中有错误信息
- 工具价格已更新
- 工具功能已变化
- 链接失效

请创建一个 [Issue](https://github.com/srxly888-creator/ai-tools-compendium/issues)。

### 2. 添加新工具

如果你想添加新的 AI 工具报告：

1. Fork 本仓库
2. 在 `reports/` 目录创建新的 Markdown 文件
3. 使用以下模板：

```markdown
# [工具名称] - 完整指南

## 概述
[工具简介]

## 核心功能
- 功能 1
- 功能 2
- 功能 3

## 定价
- 免费版：[说明]
- 付费版：[价格]

## 优势
- 优势 1
- 优势 2

## 劣势
- 劣势 1
- 劣势 2

## 适用场景
- 场景 1
- 场景 2

## 竞品对比
| 工具 | 特点 | 价格 |
|------|------|------|
| 工具 A | ... | ... |
| 工具 B | ... | ... |

## 推荐指数
⭐⭐⭐⭐⭐ (5/5)

## 相关链接
- 官网：[链接]
- 文档：[链接]
```

4. 更新 `INDEX.md`，添加新工具
5. 提交 Pull Request

### 3. 更新现有报告

如果工具已更新：

1. Fork 本仓库
2. 更新对应报告文件
3. 在文件顶部添加更新日期
4. 提交 Pull Request

## 📝 报告规范

### 文件命名
- 使用小写字母
- 用连字符分隔单词
- 示例：`ai-video-generation.md`

### 内容规范
- ✅ 客观中立
- ✅ 数据准确
- ✅ 格式统一
- ✅ 定期更新
- ❌ 避免营销语言
- ❌ 避免主观臆断

### 必需字段
每个报告必须包含：
1. **概述**：工具简介
2. **核心功能**：主要特性
3. **定价**：价格信息
4. **优劣势**：客观分析
5. **适用场景**：使用建议

## 🎯 贡献者福利

活跃贡献者将：
- 在 README 中列出
- 获得优先 Review 权
- 参与项目决策

- **Pull Requests**: [提交 PR](https://github.com/srxly888-creator/ai-tools-compendium/pulls)
