## 文章写作

### 1. 观点文章

**写作框架**
- 引入：用争议性观点或热点话题开篇
- 论点：清晰阐述核心观点（3-5个）
- 论据：用数据、案例、专家观点支撑
- 反驳：预判并回应反对意见
- 结论：总结观点，引发思考或行动

**Prompt 模板**
```
请写一篇关于[主题]的观点文章，核心论点是[你的观点]。

要求：
- 观点鲜明，立场明确
- 用3个论点支撑核心观点
- 每个论点用具体案例或数据说明
- 预判并回应2个可能的反对意见
- 结尾引发读者思考

目标读者：[读者群体]
文章风格：[犀利/温和/专业/通俗]
字数：[800-1500字]
```

**优秀案例示例**
```markdown
标题：《为什么远程工作将成为未来主流》

核心观点：远程工作不是疫情下的权宜之计，而是工作方式的必然进化。

论点1：生产力提升
- 数据：斯坦福研究显示远程工作者效率提升13%
- 案例：GitLab 全远程公司估值达27.5亿美元

论点2：人才无国界
- 招聘半径从50公里扩展到全球
- 成本降低：二线城市薪资=一线60%，但能力相同

论点3：工作生活平衡
- 平均节省1.5小时通勤时间
- 员工满意度提升40%（Buffer 2023调查）

反对意见回应：
问题：协作效率下降
回应：异步沟通工具（Notion、Lark）让协作更高效
```

**常见错误**
❌ 观点模糊，"骑墙"态度
❌ 只有论点没有论据
❌ 逻辑跳跃，缺乏过渡
❌ 忽视反对声音，显得偏激

**优化技巧**
✅ 用"但是"、"然而"制造转折，增加可读性
✅ 每段第一句是该段核心（主题句）
✅ 用对比强化观点："传统观点...最新研究..."
✅ 金句收尾，让观点传播

### 2. 深度报道

**写作框架**
- 软性开篇：人物故事或场景描写
- 背景铺陈：事件脉络和宏观背景
- 深度挖掘：多角度访谈（专家、当事人、旁观者）
- 数据支撑：统计、报告、趋势分析
- 延伸思考：影响、启示、未来走向

**Prompt 模板**
```
请写一篇关于[事件/现象]的深度报道。

背景信息：
- 事件概述：[简要描述]
- 关键人物：[主要当事人]
- 时间线：[重要节点]

要求：
- 以一个具体故事/人物开篇（500字）
- 采访3-5个不同视角的来源
- 用数据和报告支撑分析
- 揭示现象背后的深层原因
- 探讨未来影响和解决方案

风格：叙事性+分析性
字数：2000-3000字
```

**优秀案例示例**
```markdown
标题：《外卖骑手的困局：算法下的速度与生存》

开篇故事（500字）：
凌晨2点，李志送完最后一单。系统显示他今天跑了12小时，84单，收入267元。但扣除饭费、电话费、电动车折旧，实际收入不到180元。"算法就像鞭子，"他说，"你能看到它，但你跑不过它。"

背景（300字）：
中国外卖市场规模达6646亿（2023），骑手超400万。表面是自由职业，实际被困在算法里。

深度访谈：
1. 骑手视角："超时一分钟扣一半运费"
2. 平台回应："算法持续优化中"
3. 学者分析："算法变成数字泰勒主义"
4. 监管部门："正在制定新规"

数据支撑：
- 53%骑手日均工作10小时+
- 月均收入4000-8000元，无社保
- 2023年骑手相关事故上升23%

延伸思考：
这不是技术问题，是劳动权益问题。零工经济需要新规则。
```

**常见错误**
❌ 只罗列事实，缺乏叙事张力
❌ 信息源单一，只采访一方
❌ 没有数据，只有情绪
❌ 缺乏深度，停留在现象表面

**优化技巧**
✅ 用"场景化开头"：镜头感描写
✅ 多源交叉验证：至少3个不同角度
✅ 数据可视化：图表、时间线
✅ 故事+分析交替，保持节奏
✅ 结尾回到开篇人物，形成闭环

### 3. 专题研究

**写作框架**
- 研究问题：明确要解决的问题
- 文献综述：现有研究梳理
- 研究方法：定性和/或定量方法
- 数据分析：统计、访谈、案例
- 结论建议：研究发现和实践启示

**Prompt 模板**
```
请撰写一篇关于[研究主题]的专题研究报告。

研究问题：[具体问题1] [具体问题2] [具体问题3]

研究方法：
[ ] 文献综述：检索[数据库]，关键词：[关键词]
[ ] 定量研究：样本量[N]，数据来源：[来源]
[ ] 定性研究：深度访谈[N]人，焦点小组[N]组
[ ] 案例研究：分析[N]个典型案例

要求：
- 引用至少10篇学术文献
- 数据图表清晰（提供说明）
- 方法部分可复制
- 结论有实践建议

字数：3000-5000字
格式：学术规范（APA/MLA/Chicago）
```

**优秀案例示例**
```markdown
标题：《生成式AI对内容创作行业的影响研究》

研究问题：
1. 创作者如何使用AI工具？
2. AI对创作质量和效率的影响？
3. 行业未来3年趋势？

文献综述：
- 从"AI辅助"到"AI驱动"（Smith, 2023）
- 创意工作的不可替代性（Lee, 2024）
- 共15篇核心文献综述

研究方法：
定量：调研1200名内容创作者（2024年1月）
定性：深度访谈20名资深创作者

数据发现：
- 78%创作者使用AI，但60%认为AI只是辅助
- AI使用者的平均产出提升2.3倍
- 质量争议：原创性下降，但专业度上升

结论：
1. AI不会取代创作者，但会用AI的创作者会取代不会用的
2. 核心竞争力从"执行"转向"创意"和"判断"
3. 行业需要新的伦理规范

实践建议：
- 创作者：建立"AI+人类"协作流程
- 企业：投资员工AI技能培训
- 政策：制定AI内容标注标准
```

**常见错误**
❌ 研究问题不清晰
❌ 方法描述不详细，无法验证
❌ 数据缺乏来源
❌ 结论与数据不匹配
❌ 没有实践建议

**优化技巧**
✅ 研究问题用"如何"、"为什么"、"是否"等问句
✅ 方法学部分可以独立成文，便于复现
✅ 用"三角验证"：多数据源交叉验证
✅ 结论部分分"研究发现"和"实践启示"
✅ 附录：研究工具、问卷、访谈提纲

### 4. 人物专访

**写作框架**
- 开篇钩子：人物最吸引人的特质或故事
- 成长路径：关键转折点和里程碑
- 核心观点：对话式呈现，保留原声
- 个人细节：习惯、爱好、日常
- 启示总结：读者可以学到的经验

**Prompt 模板**
```
请撰写一篇对[人物姓名]的专访。

基本信息：
- 身份：[职业/头衔]
- 成就：[主要成就]
- 采访时间：[日期]
- 采访方式：[线上/线下/邮件]

核心问题框架：
1. 成长经历：最关键的3个转折点是什么？
2. 成功因素：什么让你走到今天？
3. 工作方法：你的日常流程是怎样的？
4. 挑战困境：遇到过什么困难？如何克服？
5. 未来规划：下一步目标是什么？
6. 给读者的建议：[针对特定读者群体]

要求：
- 保留人物语言风格（原声引用）
- 用具体故事支撑观点
- 穿插个人细节，让人物立体
- 引用控制在30%以内，其余为叙述

风格：[生动/专业/深度/轻松]
字数：1500-2500字
```

**优秀案例示例**
```markdown
标题：《李开复：AI时代的"教育革命"已经发生》

开篇钩子：
李开复的手机屏幕上是一张他孙女的画作，但他从不直接夸"画得真好看"。他会问："你能告诉我画里的故事吗？"这是AI时代的教育理念：提问比回答更重要。

成长路径：
- 11岁赴美：文化冲击教会他适应力
- 卡内基梅隆：AI领域先驱
- 苹果、微软、Google：从技术到管理
- 创新工场：从创业者到投资人
- AI 2.0时代：从投资到亲自下场

核心观点（对话式）：
"我见过太多聪明人失败，因为他们只解决'聪明的问题'，而不是'重要的问题'。"

"未来10年，50%的工作会被AI取代，但创造力、同理心、复杂问题解决能力这些'人'的能力会越来越值钱。"

"教育最大的失败是教会孩子标准答案。AI时代，提问能力远超回答能力。"

个人细节：
- 每天读5篇AI论文（凌晨5点）
- 周末陪孙女，从她身上学习"无界思维"
- 坚持40年晨跑，从中学会"长期主义"

读者启示：
1. 别和AI比拼"标准化能力"，那是必输的游戏
2. 投资你的"人类专属能力"：创造力、同理心
3. 终身学习不是口号，是生存必需品
4. 找到"重要的问题"，而不是"聪明的问题"
```

**常见错误**
❌ 只记录对话，没有叙述和背景
❌ 引用过多（超过50%），变成Q&A
❌ 缺乏个人细节，人物不立体
❌ 采访准备不足，问题浅层
❌ 没有明确角度，变成"大杂烩"

**优化技巧**
✅ 采访前做深度研究：社交媒体、过往访谈
✅ 用"追问"挖掘深度："为什么"、"能不能举个例子"
✅ 记录非语言信息：动作、表情、环境
✅ 每个观点配一个故事
✅ 保留人物"原声"：口头禅、表达风格
✅ 开篇用最吸引人的故事或观点

### 5. 数据新闻

**写作框架**
- 数据发现：如何发现这个数据故事
- 数据可视化：图表设计，突出重点
- 数据解读：数字背后的原因和影响
- 趋势预测：数据告诉我们的未来
- 互动设计：让读者参与探索

**Prompt 模板**
```
请创建一篇数据新闻报道。

核心数据：
- 数据来源：[官方统计/研究报告/公开数据]
- 时间范围：[起始时间-结束时间]
- 样本量：[样本规模]

关键发现：
1. [发现1：数据+变化趋势]
2. [发现2：对比分析]
3. [发现3：异常点或洞察]

要求：
- 用3-5个可视化图表支撑
- 每个图表有清晰标题和解读
- 数据标注来源和方法论
- 提供"数据背后的故事"分析
- 可选：设计互动元素（筛选器等）

视觉风格：[简洁/多彩/专业/故事化]
字数：1200-2000字（不含图表说明）
```

**优秀案例示例**
```markdown
标题：《中国房价收入比地图：哪里的年轻人买得起房？》

数据发现：
我们从国家统计局、贝壳研究院等7个来源整理了50个城市10年数据，发现一个被忽视的趋势：不是所有城市都买不起房。

数据可视化：

图表1：房价收入比热力地图
- 深圳：35.2（不吃不喝35年买一套房）
- 长沙：6.5（全国最低）
- 50城中，14城低于10年

图表2：十年变化趋势（2014-2024）
- 一线城市：从15年→28年（上涨87%）
- 新一线：从8年→13年（上涨63%）
- 三线城市：从6年→7年（上涨17%）

图表3：年轻人购房意愿
- 22-30岁：53%放弃买房（vs 2019年31%）
- "租房替代"率从18%→42%

数据解读：
1. 分化加剧：不是全国都买不起
   - 长沙模式：调控+土地供应=房价稳定
   - 深圳困境：土地稀缺+人口涌入=房价飙升

2. 代际差异：90后vs00后
   - 90后：67%认为"必须买房"
   - 00后：41%认为"租房也OK"

3. 价值观转变：从"房产=财富"到"房产=负担"

趋势预测：
- 2030年：一线城市租房率将突破50%
- "长租市场"规模将达3万亿
- 政策：共有产权房普及

互动设计：
"你的城市要工作多少年？"输入器，自动计算
```

**常见错误**
❌ 只有数据，没有故事
❌ 图表复杂，信息过载
❌ 数据来源不标注
❌ 忽视方法论和数据局限
❌ 误导性可视化（操纵Y轴等）

**优化技巧**
✅ 用"人"的故事引入数据："这个数字背后是..."
✅ 每个图表只讲一个核心观点
✅ 数据来源+方法论+局限三要素齐全
✅ 对比可视化：同比、环比、横向对比
✅ 用颜色突出重点，不是彩虹配色
✅ 移动端适配：图表在手机上清晰可读

### 6. 事件评论

**写作框架**
- 事件概述：客观描述发生了什么
- 快速反应：评论的价值和角度
- 深度分析：背后的原因、影响、趋势
- 多维视角：不同利益相关者观点
- 前瞻判断：未来发展和应对建议

**Prompt 模板**
```
请就[事件名称]写一篇评论。

事件基本信息：
- 时间：[发生时间]
- 地点：[发生地点]
- 事件概述：[200字客观描述]

评论角度（选择1-2个）：
[ ] 社会角度：对社会的影响
[ ] 经济角度：经济影响和商业逻辑
[ ] 技术角度：技术发展和趋势
[ ] 政策角度：政策制定和监管
[ ] 人性角度：个体行为和心理

要求：
- 客观描述事件（200字）
- 明确评论立场
- 分析事件背后的深层原因
- 预判未来发展和影响
- 提出建设性建议

风格：[犀利/理性/温和/建设性]
字数：1000-1500字
发布时效：[事件发生后24小时内]
```

**优秀案例示例**
```markdown
标题：《ChatGPT上线搜索功能：谷歌的噩梦才刚开始》

事件概述：
2024年1月，OpenAI在ChatGPT中推出原生搜索功能，直接挑战谷歌的搜索广告模式。用户可以在对话中获取实时信息+分析，无需跳转外部链接。

评论立场：
这不是功能升级，是范式转移。谷歌搜索的"搜索-点击-广告"模式可能被"对话-回答-订阅"模式取代。

深度分析：

1. 为什么谷歌该焦虑？
   - 搜索广告占谷歌收入80%（$1100亿/年）
   - ChatGPT搜索=无广告+精准答案
   - 用户测试：53%愿意用AI替代传统搜索（Pew Research）

2. OpenAI的战略逻辑
   - 从"工具"到"入口"
   - 订阅模式($20/月)vs 广告模式
   - 用户时长：ChatGPT平均30分钟/日，谷歌搜索10分钟

3. 用户体验革命
   - 传统：10个蓝链接+广告
   - 新模式：对话式+个性化+持续追问
   - "搜索"变成"研究助理"

多维视角：
- 用户：效率提升，但付费门槛
- 内容创作者：流量可能归零（无点击）
- 营销人：搜索广告ROI下降
- 监管：垄断担忧（OpenAI+微软）

前瞻判断：
- 2025-2027年：AI搜索份额将达30%
- 谷歌必须进化：Gemini搜索
- 新商业模式："答案即服务"

建设性建议：
- 内容创作者：从SEO转向"AI优化"(AIO)
- 企业：布局AI对话营销
- 个人：学习AI提示词，这是新搜索技能
```

**常见错误**
❌ 只有情绪，没有分析
❌ 信息错误，事实没核实
❌ 角度单一，缺乏深度
❌ 预测极端（非黑即白）
❌ 纯粹批评，没有建设性

**优化技巧**
✅ "客观描述+主观评论"明确分开
✅ 用数据支撑观点，不是"我觉得"
✅ 多维度：技术/商业/社会/人性
✅ 对比分析：历史事件、国际案例
✅ 金句传播："不是X，而是Y"
✅ 时效性：事件24-48小时内发布

### 7. 历史叙事

**写作框架**
- 时代背景：时间、地点、社会环境
- 人物塑造：主要角色的动机和冲突
- 事件铺陈：按时间线或主题展开
- 历史意义：对当时和后世的影响
- 当代启示：今天可以学到什么

**Prompt 模板**
```
请撰写一篇关于[历史事件/人物]的叙事文章。

历史基本信息：
- 时间段：[具体年份/时期]
- 地点：[国家/地区]

**优秀案例示例**
```markdown
标题：《什么是熵？为什么宇宙注定走向无序？》

类比引入：
想象你的房间：
- 整齐：每种物品有固定位置（低熵）
- 凌乱：物品随机散落（高熵）

你不需要做任何事，房间会自然变凌乱。
但你需要花力气才能让它变整齐。

这就是熵：宇宙倾向于无序。

核心原理（5步解释）：

第1步：什么是熵？
熵 = 系统的混乱程度
- 低熵 = 有序（冰块）
- 高熵 = 无序（水蒸气）

第2步：热力学第二定律
"孤立系统的熵永不减少"
翻译：宇宙的自然趋势是变混乱

第3步：为什么？
因为"无序"的状态更多。
想象3个硬币：
- 全朝上：1种可能
- 全朝下：1种可能
- 2上1下或2下1上：6种可能

无序的概率是有序的6倍！

第4步：和时间的关系
熵增 = 时间箭头
- 过去：低熵（有序）
- 现在：熵增加（变无序）
- 未来：最大熵（热寂）

第5步：生命是"逆熵"
生命是宇宙的奇迹：创造秩序
但代价：增加环境的熵
（你吃饭→维持身体有序→排泄物无序）

应用实例：
- 冰块融化：熵增加
- 你打扫房间：局部熵减少，但消耗能量（全局熵增加）
- 太阳燃烧：核聚变→能量散失→熵增加

常见误解：

❌ 误解1："熵增=混乱不可避免"
✅ 真相：开放系统可以局部减少熵（生命、空调）

❌ 误解2："熵是能量"
✅ 真相：熵是能量的"分布质量"，不是能量本身

❌ 误解3："宇宙会终结"
✅ 真相：热寂是10^100年后的事，不用焦虑

延伸学习：
- 纪录片：《宇宙的构造》
- 书：《时间简史》霍金
- 概念：麦克斯韦妖、薛定谔的猫
```

**常见错误**
❌ 过于简化，失去科学准确性
❌ 过于专业，普通人看不懂
❌ 只有概念，没有实例
❌ 忽视常见误解

**优化技巧**
✅ 类比是关键：用熟悉的事物解释陌生的
✅ "分步解释"：一步步深入，不跳跃
✅ 视觉化：用图示或比喻图
✅ 问答式：自问自答，模拟读者疑问
✅ "你知道吗"：增加趣味性

```markdown
# Notion 用户手册

## 欢迎使用Notion

Notion是你的all-in-one工作空间：
笔记、文档、数据库、项目管理，
一切都在一个地方。

## 快速入门（5分钟）

### 步骤1：创建账户

访问 [notion.so](https://notion.so)，
点击"Get Started"，
用Google或邮箱注册。

### 步骤2：创建第一个页面

点击页面任意位置，
输入标题。
恭喜！你创建了第一个页面。

### 步骤3：添加内容

Notion使用"块"(Block)概念：

- 文本：输入文字，自动创建
- 标题：输入`# ` + 空格
- 列表：输入`- ` + 空格
- 图片：拖拽或粘贴

### 步骤4：组织内容

页面可以嵌套：
- 拖拽排序
- 创建子页面
- 添加图标和封面

## 界面导览

### 侧边栏

左侧是你的工作区：
- **Workspace**: 所有页面
- **Favorites**: 收藏的页面
- **Private**: 私人页面
- **Shared**: 共享的页面

### 编辑器

中央是编辑区域：
- 顶部：页面标题、菜单
- 主体：内容块
- 右侧：评论、属性

## 功能详解

### 1. 页面

页面是Notion的基本单位。

**创建页面**：
点击侧边栏"+"，或输入`/page`

**页面设置**：
- 图标：点击页面名左侧图标
- 封面：点击顶部区域
- 属性：点击"Add property"

### 2. 数据库

Notion的强大之处。

**创建数据库**：
输入`/database`，选择类型：
- Table（表格）
- Board（看板）
- Calendar（日历）
- Gallery（图库）

**列类型**：
- 文本、数字、日期
- 单选、多选
- 人员、文件
- 公式、关联

**示例：任务管理**

| 任务 | 状态 | 优先级 | 负责人 | 截止日期 |
|------|------|--------|--------|----------|
| 设计首页 | 进行中 | 高 | Alice | 2024-02-01 |
| 编写API | 待办 | 中 | Bob | 2024-02-05 |

### 3. 模板

复用页面结构。

**使用模板**：
- Notion提供1000+模板
- 或创建自己的模板

**创建模板**：
页面菜单 → "Turn into template"

### 4. 协作

团队协作的核心功能。

**分享页面**：
点击"Share"，
选择：
- 只读
- 可编辑
- 评论

**@提及**：
输入`@`，
选择人或页面。

**评论**：
选中文字 → 右键 → "Comment"

## 常见问题

### Q: 如何导出数据？
A: 页面菜单 → "Export" → PDF/HTML/Markdown

### Q: Notion免费版够用吗？
A: 个人用户完全够用。团队建议升级。

### Q: 数据安全吗？
A: Notion使用银行级加密，SOC 2认证。

### Q: 如何迁移数据？
A: Export → Import，支持批量导入。
